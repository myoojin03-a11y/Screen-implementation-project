# 학생 상태 조회 연계 예제 프로젝트

## 프로잭트 개요
본 프로젝트는 학생들의 출석 상태(전체/ 출석 / 지각 / 결석)를 탭 클릭을 통해 필터링 되어 원하는 출석상태가 조회 가능한 예제 프로젝트 입니다.
JavaScript의 외부시스템을 이용한 배열데이터 처리, DOM조작, 이벤트 처리, 웹화면 출력를 연습하기 위한 목적의 프로젝트로 선택한 탭에 따라 학생 카드 목록과 총 인원수가 변경되어 되도록 구현하였습니다.

## 파일 구성
- `server-data.js` : 외부 학생 관리 서버에서 제공하는 학생데이터
- `index.html` : 메인 HTML파일
- `style.css` : 스타일시트
- `script.js` : 내부 처리 로직

## 실행 방법
1. 모든 파일을 같은 폴더에 저장합니다.
2. `index.html` 파일을 클릭하여 웹브라우저로 열어줍니다.
3. 필터 탭을 클릭하여 출석 상태별로 학생들의 출석 상태를 조회합니다.

## 주요 기능
- 외부데이터 (`server-data.js`)을 이용한 학생 출석 정보 로딩
- JavaScript를 통한 학생 카드 목록 동적 생성
- 출석 / 지각 / 결석 상태에 따른 필터링 기능
- 선택된 필터 탭에 따른 총 학생 수 실시간 표시
- 출석 상태값에 따라 라벨 색상 자동 변경

## 시험 요구사항 설명
### 1. 외부 서버란 무엇인가?
외부 서버란 현재 웹 페이지 외부에서 데이터를 제공하는 시스템을 의미하며,
본 프로젝트에서는 실제 서버 환경을 가정하여
`server-data.js` 파일에 학생 출석 데이터를 분리하여 관리하였다.

이는 실무에서 API 또는 서버로부터 데이터를 받아오는 구조를
학습하기 위한 목적이다.

---

### 2. 연계 데이터란 무엇인가?
연계 데이터란 외부 시스템에서 제공되어
내부 웹 페이지에서 가공 및 활용되는 데이터를 의미한다.

본 프로젝트에서는 학생의 ID, 이름, 클래스, 출석 상태 정보를
연계 데이터로 사용하였으며,
해당 데이터는 카드 UI 생성 및 필터 기능에 활용된다.

---

### 3. 외부 서버 -> 내부 웹데이터 흐름
외부 데이터는 `server-data.js`에서 배열 형태로 관리되며, 내부 JavaScript 파일 `script.js`)에서 이를 불러와 다음과 같은 흐름으로 처리된다.

1. 외부 데이터 로딩
2. 학생 정보를 기반으로 카드 UI 동적 생성
3. 사용자 탭 클릭 이벤트 발생
4. 출석 상태에 따른 데이터 필터링
5. 화면과 총 인원 수 갱신

---

### 4. 왜 이 구조가 연계구조인가?
외부 데이터 파일과 내부 UI 처리 로직을 분리함으로써 데이터 변경 시 화면 로직을 수정하지 않아도되도록 설계하였다.
이는 실제 웹 서비스에서 서버와 프론트엔드가
서로 독립적으로 동작하는 구조를 모방한 연계 구조이다.

---

### 5. 본 예제로 평가 가능한 능력 단위
본 프로젝트를 통해 다음과 같은 프론트엔드 개발 능력을 평가할 수 있다.

- 외부 데이터 구조 이해 및 활용 능력
- JavaScript 배열 및 객체 처리 능력
- DOM 조작을 통한 화면 출력 구현 능력
- 이벤트 기반 사용자 인터랙션 처리 능력
- 조건 분기를 이용한 상태별 UI 제어 능력
